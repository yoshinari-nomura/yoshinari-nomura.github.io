私は、こちらの ssh は使ったことがなく、また、

山下> [/home]mhc-sync -x neptune takay@terra
山下> Initalizing local ... done.
山下> Connecting terra ...
山下> takay@terra's password:
山下> Initalizing remote (exchange_id=neptune) ... done.

のように山下さんは ssh-agent をお使いになっていないようなので、
よろしかったら、以下のことを実験していただけるとありがたいです。

背景:
Ruby の mhc-sync を Elisp から動かす mhc-sync.el というのが
current にあるのですが、Win32 上で山下さんの使い方の様に
password あるいは passphrase を入力するときに Elisp から ssh に
passwd がきちんと伝わるかわかっていません。

# ちなみに私の使っている某所から入手できる ssh は × です。もし、
# 山下さんの使われている ssh が ○ なら、ちょっとうれしいのです。

方法:
mhc-sync.el をちらっと見ていただいて、~/.emacs に
% mhc-sync -x neptune takay@terra なら

(setq mhc-sync-id "neptune")
(setq mhc-sync-remote "takay@terra")

の設定をします。そして、mhc の立ち上がっている状態で

M-x mhc-file-sync

です。
何事もなく終わってしまうなら M-x mhc-file-toggle-offline が必要
かもしれません。
そうすると、buffer が立ち上がって、

Initalizing local ... done.
Connecting terra ...
takay@terra's password: <== このときに

mini-buffer に password を入力するように求められますので、打ち込
んでリターン。

で、動くかどうか、です。(実際に転送する schedule はあってもなく
てもかまいません。Initalizing remote ... done. が出れば OK です。

# 最悪、Meadow が固まるかもしれないのですが許してくださいませ。

というわけで、よろしくお願いします。(_ _)
