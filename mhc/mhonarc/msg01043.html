> ＃ Secret か Hidden とどっちがいいんだろうか…．

えっと、僕の中では、Secret (現Private) と Hidden は別の考えで、

  Hidden : 存在を知らせないように隠す
  Secret : 存在は分かるようにするけど、それが何あるかは、
	   Summary を見ただけでは分からない。

TODO でいうと、Secret の場合は、一覧に 「野暮用」 と出る。
Hidden は、一覧にすら出ない。ので、なかやまさんのイメージ
と重なるのは、僕のいう、Hidden ですね。
2つの言葉がそれぞれに似つかわしいかどうかは
僕にはよく分かりません。ただ、
少なくともこの 2つの概念を区別したいです。

> そこまで自由度上げてもややこしさが増えるとおもうです．
> そこで，のむらさんの方法で行くなら，Todo を消すときは
> Todo を OldTodo って変えて，OldTodo を Todo list に表示するか
> しないかを切り替えできるように指定できるってだけでよいのでは
> ないでしょうか…とかおもったのですが，

スケジュールで使うカテゴリとの直交性が悪くなるので、
Todo を OldTodo に変えてではなくて 「加えて」 にしました。

> でも，それだと，結局 OldTodo が貯まっていってしまうだけで，あんまり
> それ（OldTodo なやつ）を見ることも無い気がするので，どうせならやっぱ
> Todo Done を Done に変えて，日付無しはそうすることを禁止してしまうっ
> て風のが単純でいいような気もしてきました．

別の見方をすると、

  「Category: Todo と書いてある物だけを Todo だとみなします。」★

という方が単純で、Done 単体がある場合の方が面倒に見えてくるのは
僕だけでしょうか。

★の原則がまず第一で、より限定的な意味を与えたければ、
別の属性を足すのが正しいと思います。
なので、OldTodo を付けるとすれば、TODO に並べて、
OldTodo と書くのが正しい。もっというと、OldTODO というのは、
二重に物事を限定しているので、Hidden とかにするのはどうかということです。

インターフェースは全然変わらないと思いますよ。
説明もたぶんこの方が楽だと思います。
