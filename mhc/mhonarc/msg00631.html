乃村です。

最近の contribute を取り込むのは 0.25 が出してからゆっくり、
と思っていましたが、ぐずぐずしていると冷めて美味しくなくなって
しまいそうなんで、枝を用意しました。

 % cvs rtag -b RELENG_00 mhc 

0.25 に向けての作業は、この枝に対して行われます。
安定バージョンをお望みの方は、こちらから checkout しなおして下さい。

  cvs -d :pserver:anonymous@xxxxxxxxxxxxxxxxx:/cvsroot login
  CVS password: <- そのまま CR
  cvs -d :pserver:anonymous@xxxxxxxxxxxxxxxxx:/cvsroot co -r RELENG_00 mhc

今後、

1. 幹には常に最新の変更をどんどん入れていく。
   そのため、一時的に動作しない snap も発生することがある。

2. 適当な時点で feature freeze を行い、RELENG_xx なる枝を
   作成して、コードをテスト, デバッグし、ドキュメントを書く。

   xx はメジャーバージョンを表わす 2桁の数字。
   例えば 0.25 では 00, 3.30 では 03 とする。

3. RELENG_xx 枝からのリリース時点には RELEASE_xxyy なる tag を貼る。

   xxyy はメジャー・マイナーバージョンを表わす 4桁の数字。
   例えば 0.25 では 0025, 3.30 では 0330 とする。

のような方針で行こうと思っています。この規則にのっとると、

  RELENG_00 なる枝を作成する (done)
  RELENG_00 を安定化させて、RELEASE_0025 なる TAG を貼ってリリース (soon)
  幹には committer による変更ががんがん入る (soon)
  (頃合を見て)
  RELENG_01 なる枝を作成する (further away)
  :
  以下繰り返し

となると思います。よって、0.25 の次の枝は、自然に 1.00 候補になります。:-)
