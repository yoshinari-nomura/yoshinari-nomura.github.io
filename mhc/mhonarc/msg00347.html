白井> 2. 同一 buffer に縦型カレンダを表示すると、表示が乱れる。
白井>  (tab=8 で見てね)

Emacs 19.28/Mule 2.3 でも同様になりました(というかもっとひどい
(*1))。結論からいうと、手持ちで調べた限りでは、

行頭の最初に visible なところ(*2)の (current-column) の値が

Emacs 20.4, 5 => '0'
XEmacs 21.2, Emacs 19.28 => Mew なら '4', wl なら '9'

となるために *見た目* がくるってしまっているようです。

というわけで、とっても汚いし遅くなってしまったのですが、上記の問
題に対応しました。
source をいじり倒してしまい、なにがなんだかわからなくなってしまっ
たので、snap20000112 からの diff です。*別バッファ縦型カレンダ*
や what-line() のパッチも含みます。

# もっとすっきりできないかなぁ。

*別バッファ縦型カレンダ* の既知の問題点としては、なるだけ邪魔に
ならないように post-command-hook で縦型カレンダを消しているので
すが、こんなコストの高いフックを使っていいのか？ とか、XEmacs で
は "g" を押してフォルダの一覧を出すと縦型バッファに表示されてし
まって寂しい。とかいろいろあるので、もう少し考えないといけないよ
うです。^^;

注釈)
(*1)
そういえば、そうだったんですが、19.28 附属の insert-rectangle は
短い行の行末に空白を入れてくれない。

(*2)
01/01  Sat             元日
