MHC のディストリビューションについてですが、こうだったら配布する場合に
いいなあ、という基本方針を以下に述べさせていただきます。
ご検討ください。
要は、copyright ごとに分けて配布していただけたらありがたいということで
す。

 1. copyright が違うアドオンは contrib ディレクトリー以下に配置する。
 2. MHC のメインディストリビューションを適当なディレクトリーに配置する。
 3. experimental なコードは experimetal ディレクトリー以下に配置する。
 4. 英語のコピーライト文書は必須。(英語でないと U.S. の連中が Freesoft
    かどうかを判断できないためです。もちろん、オリジナルが日本語で、英
    語は飜訳でもいいです。英語がオリジナルである必要はありません。)
 5. 英語のドキュメントは必須ではない。
 6. (できればの希望) Makefile を添付していただくとありがたいです。

なお実際に Debian パッケージとなる場合の方針は以下のようになる予定です。

 1. copyright がついていないソフトウェアはバイナリー、ソースとも配布か
    ら外します。
 2. experimental で copyright がついているものはソースパッケージのみ
    の配布とします。

配布にあたって一番重要なのが、copyright がしっかり規定されていることで
す。

Debian GNU/Linux のコピーライトに関する規定は、おそらくフリーソフト界
随一に厳しいです。これは、著者の権利を守るためでもあり、真のフリーソフ
トを守るためでもあります。また、配布してはいけないものを Debian が配布
しないためでもあります。

Debian の基本方針として、copyright がついていないソフトウェアは配布で
きないことになっています。これは、copyright が恣意的に変更されてしまう
可能性があるからです。

詳しくは DFSG (Debian Free Software Guideline) をごらんください。
http://www.debian.org/social_contract#guidelines
# Contents-negociation で 日本語が読めるようになっています。

これは Open Source Definition のもとになった文書で、opensource.org で
Opensource Definition を書いた Bruce Perens は Debian のプロジェクトリー
ダでした。
http://www.opensource.org/osd.html

GPL や BSD style、Artistitc、X11 などは DFSG に該当しているフリーソフ
トのライセンスとなっています。

ただし、ウェブページには明示的には書いてありませんが、BSD スタイルライ
センスについては、Not Recommend だったりします。

BSD がなぜ Not Recommend かは、八田真行さんの以下の文書を参照してくだ
さい。Richard Stallman の文章の飜訳です。
http://www.geocities.co.jp/SiliconValley-PaloAlto/9803/translations/bsd-ja.html

#もちろん、これは Debian の事情の説明であって、乃村さんにライセンス変
#更をせまっているわけではありません。

DFSG の骨子を抜きだしておきます。

* ソースが入手できなくてはならない。

* 自由な再配布、変更ができなくてはならない。

* ライセンスは Debian に限定されない。-> Debian からパッケージを抜きだ
  して再配布してもライセンスがかわってはいけない。

* 配布対象によって差別してはならない。

お忙しいところすみませんが、よろしくご検討ください。
