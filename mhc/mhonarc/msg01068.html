馬場＠京大宇宙物理 です。

mule-19.34.1 + mew-1.94.2 という環境で、mhc-cvs を使ってみているの
ですが、いくつか問題があるようです。

まず APEL を入れて ~/.emacs で (load "tinycustom" t t nil) などと
しておかないと "Symbol's function definition is void: defgroup" に
なります(これでいいのかわかりませんが、とりあえず動いてはいる)。

次に、スケジュールの新規追加や削除はできますが、内容を変更しようと
すると "Cannot get schedule slot" となってしまい、変更できません。

また、$CVSROOT が :ext:user@host:/path/to/repository タイプのとき
は、リモートマシンの $CVSROOT/schedule から checkout してくれません。
とりあえず最初だけ手動で

  % cd ~/Mail/
  % cvs -d :ext:user@host:/path/to/repository co schedule

しておいてから、スケジュールを追加・削除しても Sync がうまく働きま
せん。つまり、~/Mail/schedule/.mhc-db-log には A とか D とかいうマー
クは付いているのですが、リモートマシンに cvs commit してくれません
(もちろん ONLINE 状態で)。

これらは、どのへんをいじれば直せるでしょうか?
